          # Linux文件特殊权限

          查看操作下同下的/tmp文件，可以看到权限如下：

          $ ls /
drwxrwxrwt.  12 root root  4096 12月 24 15:53 tmp
          1
          2

          我们常见的权限通常为rwx，当x位出现其他字母时，代表使用了特殊权限。

          了解特殊权限需要先三个概念：SUID、SGID、SBIT。

          1. SUID：属于user位，对二进制可执行文件生效，代表只允许所有者以及超级用户有执行权限，

            赋予该权限：chmod u+s /xxx。

          2. SGID：属于group位，对文件及文件夹都生效，代表只允许组用户以及超级用户有执行权限，

            在该文件夹下创建的文件属于该文件组，赋予该权限：chmod g+s /xxx。

          3. SBIT：属于other位，只对文件夹设置，在该文件夹下创建的文件以及文件夹只允许所有者以及超级用户有删除权限，赋予权限：chmod o+t /xxx。
